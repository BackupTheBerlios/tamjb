What Jukebox?

Track Attribute Management JukeBox (Tamjb) is a music player that randomly picks songs to play, based on how much you think they suck.

It Makes No Assumptions

  • It considers every track to be in every playlist until you say otherwise.
  • It chooses tracks with a relative frequency depending on how much you think they suck.
  • If a song sucks, you push the button, and the jukebox learns a valuable lesson about your opinion.

A Story About The Big Button Labeled Suck

The first user interface had just one button, the Suck button. And it was good, but it was not quite complete. Sometimes a song did not suck, but was merely wrong for the current mood. Sometimes the jukebox was listened to by different people. So, the concept of mood and user were added, and it was better.

Because You Can't Be Negative All The Time

It became obvious that some of these moods were actually not moods, for example "rockin" or "Downtempo". And the listeners, because they were end users of the software and therefore irrational and demanding, expressed a wish to combine several moods at once.
But the programmers did not wish to add these features yet, because it was "difficult". As a compromise, the moods were stored in the database as track attributes, and managed by the Track Attribute Manager.
And so, in fire and hot water, TAM is born.

Some Features

  • It plays MP3 format files at 44.1kHz. And nothing else at this time.
  • Reads song information from ID3V2 tags.
  • Web interface allows remote control of your jukebox from anywhere in the house (or work, if you're streaming to yourself somehow.)
  • Multi-band leveling with limiting and soft clipping. Linear-phase FIR crossovers for minimal artifacts. It sounds good, but boy does it use CPU!
  • Allows multiple users to log in, and tries to please them all. Note that if too many people log in, this means you get boring music. (This is also how modern radio stations work!)

System Requirements

If you have a modern linux, you can probably run the jukebox. All development is now done using Debian. WindowsTM is not currently supported, because TamJB uses Esound for audio output. Someday GStreamer will be complete, and a Windows port may become available. (And Duke Nukem Forever will ship, too.) What I'm saying is, you want a Debian system with some stock packages, or any other Linux and the ability to install software without sending annoying emails to the author.
Hardware-wise, it will run on a Pentium 3. However, you'll have to turn off the more advanced leveling features, and your machine will work pretty hard. An AMD64 3000+ is adequate. You can also disable most of the DSP and run it on a slower system, but you'll want to use some sort of outboard audio equalization.
